使用Gridea搭建个人博客（一）：安装、主题及部署至Github

Author: SamsonMei Date: 2020-01-01 Reading Time:5.8 mins words:1595 Category: 博客搭建
作者: SamsonMei 日期: 2020-01-01 阅读时间:5.8 分钟 字数:1595 分类: 博客搭建

搭建个人博客的教程网上已经有很多了，Gridea的教程看官方文档也足够了。这篇blog更多的是记录今天折腾了大半天的成果，以及留着以后备查吧~

前言

之前也不是一个擅长输出的人，上学的时候，同学们往往在QQ空间、校内网、网易博客、百度博客等平台写字，而我从来没有下决心在哪个地方写点什么，虽然也随波逐流的注册了一些平台，偶尔写些不痛不痒的短文。真正想着找一块自己的地方还是在2017年，换了工作、换了生活的城市，用Hexo+Github搭建了个人博客，但却没有坚持下来。当时沉迷于设置模板，NexT主题设置花费了不少时间，真正文章却没写多少……

只有输出，才能打造学习的闭环；只有输出，才能说明真正掌握了所学的内容。本着这个理念，打算再次开通一次个人博客，试着写点东西。这次没有选择Hexo，还是因为Hexo生成文章确实太慢了，之前我没有几篇文章时生成网站就需要用几秒了；后来听说了Bitcron，然而是邀请制，我实在不想为了买一个自己用不到的markdown编辑器浪费钱；在研究Hugo的时候，无意中发现了Gridea，图形化的界面一下就吸引到了我，毕竟能够减少命令行的使用，更多的简单化操作，痛苦度会更低一些。于是就选定了Gridea，折腾了大半年，算是有了这个简陋的博客（并不是说Gridea以及主题简陋，主要是我并没有太认真的设置）。下面，开始吧~

安装Git以及Gridea

安装Git

要想远程部署在Github或者Coding，必须要安装Git
我个人的环境是macOS，所以安装很简单：brew install git

安装Gridea

前往Gridea官网，下载对应版本的安装包，安装即可

主题配置

安装好后，双击图标打开，软件的界面大概是这个样子：

点击红框内“主题”，进入主题配置界面：

这里我们一项项展开说：

基础配置

选择主题

Gridea自带有4款主题：Notes Simple Fly Paper，官网有主题市场，可以去下载使用，安装也很简单，下载后解压至 文件夹\Gridea\themes ，然后在第一项“选择主题”中选中即可。这里我选择了付费主题 Walker，6元并不贵，就算是支持Gridea的开发者啦~

网站名称

顾名思义，就是我们打算搭建的个人博客名称啦，起一个喜欢的名字吧！

网站描述

类似于副标题，简述网站性质、博主性格，或者是一句格言也可以，我在这里使用了喜欢的一句格言。需要注意的是，如果描述文字比较长，Gridea生成的页面不会按照标点或者配置这里的换行来显示，会按照预定的页面宽度截断，所以可能会比较难看。这里有一个小窍门，在需要换行的位置增加一个HTML标签<br>，就可以实现换行效果。

网页图标和头像配置

一个是在浏览器中打开网页时，在最左侧显示的图标，一个是博客首页的个人头像，按需设置即可

预览

主题配置好后，可按左下方“预览”按钮，在浏览器中查看博客网页实际效果

远程配置

要想把自己的博客发布到互联网上，就需要进行远程配置了。Gridea提供了Github Coding SFTP三种方式，我是部署在Github上的，Coding也类似，这里就以Github为例来介绍。

注册及配置Github

  1. 前往Github注册账号
  2. 创建好账号后，选择网页右上角+号，选择New repository，建立新仓库
  3. 按照下图对新仓库进行设置：

    需要注意的是，仓库名必需为用户名.github.io，这是github的规定
  4. 点击这里创建一个Token：

    只需要选上repo，然后点击下方的Generate token

    把Token保存下来，因为这个页面刷新后就不再显示Token了

Gridea配置

进入远程配置界面，对照上面Github的配置，填写有关信息
需要注意的是，域名前要加上https://；分支为master
保存后，点击“检测远程连接”，没有问题的话，点击左侧“同步”就部署到远程啦~
